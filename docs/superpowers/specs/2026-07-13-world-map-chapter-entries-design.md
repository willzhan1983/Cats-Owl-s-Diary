# 世界地图篇章独立入口设计

## 目标

让世界地图上的四个已开放篇章都拥有一个明确、唯一的进入按钮：

- 森林学校
- 月光湖
- 苹果谷
- 森林公路

未开放且没有可玩关卡的区域继续保持锁定，不显示进入按钮。

## 实现范围

沿用当前按篇章拆分的入口脚本，不重构 `world-map.js`：

- 保留 `moonlight-map-entry.js`
- 保留 `apple-valley-map-entry.js`
- 新增 `forest-school-map-entry.js`
- 新增 `forest-road-map-entry.js`
- 在 `index.html` 中加载四个入口脚本

每个脚本只负责对应区域：玩家在世界地图选中区域后，详情面板显示一个 `进入××篇` 按钮。重复选择区域时不能生成重复按钮。

## 进入行为

- 森林学校从 Day 1 开始。
- 月光湖从第一个 `world === "moonlight_lake"` 的关卡开始。
- 苹果谷从第一个 `world === "apple_valley"` 的关卡开始。
- 森林公路从第一个 `world === "forest_road"` 的关卡开始。
- 点击入口后关闭世界地图、进入对应首关并显示原有开始提示。
- 图片或入口脚本异常时不改动关卡数据，不造成白屏。

## 不修改

- 不修改 Day 1-7 关卡内容。
- 不修改月光湖、苹果谷、森林公路玩法。
- 不修改积分、结算、题库、难度或个人资料系统。
- 不为橡果镇、河畔码头、湿地公园、迷雾沼泽、星光山增加假入口。
- 不合并或重构四个入口脚本。

## 验证

- 自动检查四个入口脚本均由 `index.html` 加载。
- 自动检查四个入口按钮文案和对应首关跳转逻辑。
- 浏览器检查四个已开放区域各显示一个入口按钮。
- 浏览器检查锁定区域不显示入口按钮。
- 运行相关 Node 语法检查与 `git diff --check`。
